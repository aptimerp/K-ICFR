## 개요
금강공업 내부회계관리제도(ICFR, Internal Control over Financial Reporting)에 관련된 통제 규칙 및 업무 규칙.

---

## 1. 전표 관련 규칙

### 1-1. 전표 마감 후 수정 금지
- **규칙**: 월/분기 말 마감된 전표는 수정 불가
- **예외**: CFO, 감시 담당자 권한 (감시자만)
- **구현**: 
  - 마감된 기간 전표 UPDATE 시 에러 반환
  - 감시자 권한 체크 필수

```sql
-- 예: Supabase 정책
CREATE POLICY "vouchers_cannot_be_modified_after_close" 
ON erp_vouchers
FOR UPDATE
USING (
  (SELECT period_closed FROM periods 
   WHERE period_date = vouchers.voucher_date) = false
  OR current_user_role = 'AUDITOR'
);
```

### 1-2. 시간차 로깅 (감시용)
- **규칙**: 매입·매출 분개 시간 기록 (SEC 규정)
- **기록 항목**:
  - 거래 발생 시각
  - 분개 입력 시각
  - 승인 시각
- **구현**: 전표 생성/수정/승인 시 `created_at`, `updated_at`, `approved_at` 타임스탬프 자동 기록

### 1-3. 다단계 승인 (결재 규칙)
- **₩10M 이상**: 과장 → 부장 → 이사 순차 승인
- **₩1M~₩10M**: 과장 → 부장 승인
- **₩1M 미만**: 과장 승인
- **예외**: CEO, CFO는 자동 승인

```
IF 금액 >= 10,000,000 THEN
  Required Approvers = [Manager(과장), Director(부장), Executive(이사)]
ELIF 금액 >= 1,000,000 THEN
  Required Approvers = [Manager, Director]
ELSE
  Required Approvers = [Manager]
END
```

### 1-4. 거래 번호 자동 부여
- **형식**: `YYYY-MM-DD-HHMM-SEQ` (예: `2026-05-15-1430-0042`)
- **규칙**: 거래일 기준 일련번호 (중복 방지)
- **구현**: 전표 생성 시 DB 시퀀스 활용

## 2. 수불 관련 규칙

### 2-1. FIFO 원칙 (선입선출)
- **규칙**: 평균원가법 미사용, FIFO 엄격 적용
- **관리**: 입고 시간 기반 출고 순서 결정
- **예외**: 부패·손상·쓸모없어진 물품 (LIFO 허용, 문서화 필수)

```
출고 = 가장 오래된 입고 로트부터 선택
예: 
  - 1월 입고 100개 @ ₩10/개 = ₩1,000
  - 2월 입고 50개 @ ₩12/개 = ₩600
  → 3월 출고 120개: 1월 100개 + 2월 20개 = ₩1,240
```

### 2-2. Zero-stock 점검
- **주기**: 매월 2일 09:00 ~ 12:00
- **대상**: 전 창고 (본사, 언양, 부산)
- **산출물**: 재고 실사 보고서 + 차이분석
- **처리**: 
  - 손상·분실: 손실 전표 즉시 작성
  - 초과: 선입선출(FIFO) 재점검

### 2-3. 언양 공장 분리
- **독립 수불**: 언양 공장은 별도 수불 시스템 운용
- **월간 대사**: 본사 ↔ 언양 월 1회 재고 대사 (매월 5일)
- **차이 조정**: 대사 미해결 시 경영진 보고 (₩100K 이상)

## 3. 내부회계관리평가 규칙

### 3-1. 평가 체크리스트
각 평가마다 다음 항목 검증:

| 항목 | 요구사항 | 증거자료 |
|-----|--------|--------|
| **정책 문서화** | 제어 활동 정의서 | 규정집 또는 운영 메뉴얼 |
| **책임 명시** | 담당자 이름·직급 | 서명된 위임장 또는 구성도 |
| **수행 증거** | 실행 기록 (월 1회 이상) | 점검표·실행 기록 |
| **효과성** | 제어가 실제 작동하는가? | 점검 보고서, 예외 처리 사례 |
| **예외 대응** | 위반 시 즉시 조치 계획 | 시정조치 기록 |

### 3-2. 위험 분류
| 가능성 (L) | 영향도 (I) | 잔존 위험 | 조치 |
|----------|----------|---------|------|
| HIGH (L=3) | HIGH (I=3) | 9 (극대) | 🔴 즉시 시정 |
| HIGH, MEDIUM | MEDIUM | 6, 8 (높음) | 🟠 30일 내 완료 |
| LOW, MEDIUM | LOW | 1~4 (낮음) | 🟡 모니터링 |

## 5. 데이터 보존 & 감시

### 5-1. 기록 유지 정책
- **평가 기록**: 최소 5년 보관 (법규 준수)
- **감사 흔적**: 전체 변경 이력 로깅 (수정 불가)
- **백업**: 일일 백업, 분기별 오프사이트 아카이브

### 5-2. 감시자 감사 로그
아래 항목은 감시자만 접근:
- 마감 후 수정된 전표
- 역권한으로 승인된 거래
- 다단계 승인 규칙 위반 건

---

## 6. 예외 & 면책

### 6-1. 긴급 거래
- **정의**: 영업 일정 관계로 정상 절차 위반 필수
- **승인**: CEO + CFO + 감시담당자 사전 동의
- **보고**: 월말 예외 거래 리스트 (이사회 보고)
- **예시**: 급작스러운 수주 → 신용조건 미흡 → CEO 특별 승인

### 6-2. 시스템 장애
- **대응**: 수동 기록 (별지 양식) → 장애 해결 후 시스템 입력
- **보고**: 1시간 이상 장애 → 경영진 보고

## 7. 컴플라이언스 (법규)

### 내부회계관리제도 (ICFR)
- **법적 근거**: 자본시장법 § 80-1
- **평가 주기**: 분기별 / 연 1회 종합평가
- **보고**: CEO·CFO 보증 필수

### 감사·공시
- **외부 감사자**: 연 1회 감시 평가
- **공시**: 사업보고서 내 "내부통제 실태" 기재

## 8. AI 시스템 구현 시 주의

### 8-1. 자동화 규칙
- ✅ **자동화 가능**: 평가 체크리스트 생성, 위험 분류, 알림
- ❌ **자동화 불가**: 승인(최종 판단은 인간), 마감(CFO 최종 결정)

### 8-2. 감시 기능
- 변경 로그 자동 저장 (삭제 불가)
- 규칙 위반 경고 (감시자에게 즉시 보고)
- 분기별 컴플라이언스 리포트 자동 생성
