## 작업 분해 (T1 ~ T7)

### T1. 공통 사이드패널 컴포넌트 `ApprovalDrawer.jsx` 신규 생성
- **작업**: 우측 슬라이드 Drawer 공통 컴포넌트
  - props: `open`, `onClose`, `title`, `children`
  - 오버레이 클릭·ESC 닫기, 우측 슬라이드 트랜지션
  - 내부 영역: 증빙 미리보기 슬롯 + 의견 입력 슬롯 + 액션 버튼 슬롯
- **산출물**: `components/ApprovalDrawer.jsx`
- **검증**: import 시 빌드 오류 없음, open=true 시 패널 노출
- **의존성**: 없음 (가장 먼저)

### T2. 결재관리 메뉴에서 "내 할일" 제거 (M2)
- **작업**: `components/AppLayout.jsx` approval children에서 `/approval/todo` 라인 삭제 → 상신함·결재함 2개만
- **산출물**: `components/AppLayout.jsx` (수정)
- **검증**: 사이드바 결재관리 하위 2개만 노출
- **의존성**: 없음

### T3. `/approval/todo` 페이지 삭제 (M2)
- **작업**: `app/approval/todo/page.jsx` 파일 삭제 (라우팅 제거)
- **산출물**: 파일 삭제
- **검증**: `npm run build` 시 todo 라우트 미생성, 깨진 링크 없음
- **의존성**: T2 (메뉴 링크 먼저 제거 후 삭제)

### T4. 홈 "내 할일" = 요약 + 딥링크로 전환 (M1)
- **작업**: `app/page.jsx`
  - "내 할일" 작은 카드 → **유형별 건수 요약** 표시 (증빙제출 N · 승인대기 N · 반려 N)
  - 클릭 시 `/approval/todo`가 아니라 **해당 작업 탭**으로 딥링크 (증빙=op-eval/evidence, 승인=approval/inbox 등)
  - 5개 페르소나 홈(P1~P5) 모두 점검 — todo 링크 잔존 제거
- **산출물**: `app/page.jsx` (수정)
- **검증**: 홈에서 todo 링크 0건, 유형별 건수 요약 노출
- **의존성**: T3 (todo 제거 후 링크 재배선)

### T5. 상신함 디테일 구현
- **작업**: `app/approval/sent/page.jsx`
  - 컬럼 추가: **현재 결재자** (예: "이차장 결재 대기")
  - 행 클릭 → ApprovalDrawer: **증빙제출 의견** 표시
  - 체크박스 + **일괄 취소** 버튼 (결재중 상태만 선택 가능)
  - **재상신 버튼** (반려 상태 행) → Drawer에서 수정·재상신 흐름 (더미)
- **산출물**: `app/approval/sent/page.jsx` (수정)
- **검증**: 4개 항목 동작, 빌드 통과
- **의존성**: T1 (Drawer)

### T6. 결재함 디테일 구현
- **작업**: `app/approval/inbox/page.jsx`
  - **통제번호 클릭 → ApprovalDrawer**: 상신자 증빙 원본 조회 + **증빙 미리보기**(더미 파일/이미지)
  - **단건 의견 입력** 칸 (Drawer 내)
  - **반려 시 의견 필수**: 의견 비어있으면 반려 버튼 비활성/차단 + 안내
  - **승인 일시 표시**: 승인 처리 시 시각 기록·노출 (더미 상태)
  - 기존 일괄 승인/반려 유지
- **산출물**: `app/approval/inbox/page.jsx` (수정)
- **검증**: 5개 항목 동작, 반려 의견 미입력 차단 확인
- **의존성**: T1 (Drawer)

### T7. 빌드 검증 + 문서 마킹
- **작업**:
  - `npm run build` 실행 (경고 0건 목표)
  - 깨진 링크·미사용 import 정리
  - `.planning/STATE.md`·`ROADMAP.md` Phase 8-B 진행 마킹
- **산출물**: 빌드 성공 로그, 문서 갱신
- **검증**: 전 페이지 정적 생성 성공, todo 라우트 제거 확인
- **의존성**: T1~T6 전체
